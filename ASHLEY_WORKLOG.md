## 第 1 輪：首頁手機版對稱修正

### 問題
- 手機版首頁 header 的 logo 容器寬度與實際圖片寬度不一致，造成頭像視覺中心偏右，整個首屏看起來不對稱。

### 修改
- 調整 `assets/css/main.css` 中手機版 `#header .image.avatar` 與 `#header .image.avatar img` 的尺寸規則。
- 讓容器與圖片同步為 `100px × 100px`，並以 `margin: 0 auto` 保持真正置中。
- 補上 `object-fit: cover`，避免未來圖片比例改變時再次歪斜。

### 驗證結果
- 使用 Playwright headless 重新讀取 iPhone 12 視窗下的 layout。
- 修改前：avatar 容器寬 `80px`，圖片寬 `100px`，圖片右側外溢，中心偏移。
- 修改後：avatar 容器與圖片皆為 `100px`，`x = 145`、`width = 100`，左右對稱。
- 本輪修改有效，首屏視覺中心已明顯穩定。

### 學到的規律
- 手機版不對稱不一定來自文字，常常是「容器尺寸」和「內容尺寸」不一致造成的視覺偏移。
- 檢查 `getBoundingClientRect()` 比單看 CSS 更可靠，能直接看出是否真的置中。
- 這個站目前有多層 media query 疊加，修手機版時要特別小心後面的覆寫規則。

### 後續建議
- 下一輪優先檢查首頁 header 文字換行與導覽間距，因為首屏雖已置中，但文字節奏仍偏緊。
